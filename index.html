06 — Рефлексия

Менің тәжірибем

01

Ең пайдалы құрал

Осы жұмысты орындауда маған ең көп Claude көмектесті. Оның ұзын контекстті есте сақтай алу қабілеті және кодты нақты, қадамдап түсіндіруі — HTML/CSS/JS үйренушілер үшін өте қолайлы. Сонымен қатар, GitHub Copilot IDE-де жазған кезде тікелей ұсыныс беруі уақытты үнемдетті.

02

ЖИ нені дұрыс / ыңғайсыз жасады

ЖИ CSS Grid мен Flexbox-ты тамаша жасады — бірақ кейде стильдер бір-бірімен қайшы келді. Мен оны DevTools арқылы тексеріп, нақты элементтерді өзім реттедім. ЖИ «бәрін бір жолмен» жасауға тырысады, ал нақты дизайн — итеративті процесс екенін түсіндім.

03

ЖИ 5 жылдан кейін бағдарламашыны қалай өзгертеді?

Менің ойымша, 5 жылдан кейін бағдарламашы рутиналық код жазудан алшақтап, архитектуралық шешімдер мен бизнес логикасына назар аударады. ЖИ — «код генераторы» емес, «идеяны прототипке айналдыратын серік» болады. Бірақ debugging, security аудит және жүйені терең түсіну — адам бағдарламашының негізгі артықшылығы болып қала береді. ЖИ бізді алмастырмайды — бізді күшейтеді.
